What is Hermes?

An open-source document management system for creating, reviewing, and publishing technical documents.

Document Types

  • RFC (Request for Comments): Technical proposals
  • ADR (Architecture Decision Record): Design decisions
  • MEMO: Analysis, guides, and documentation
  • PRD (Product Requirements Document): Feature specs

Core Features

  • Create and edit documents with structured templates
  • Review and approval workflows
  • Full-text and semantic search
  • Multiple storage backends (local files, Google Workspace)

Built by HashiCorp

Created to scale the document writing process at HashiCorp, now open-sourced for teams that need flexible, self-hosted document management.

Configuration

Swap backends without code changes

Local Development


providers {
  auth      = "dex"          # Local OIDC
  workspace = "local"        # Markdown files
  search    = "meilisearch"  # Self-hosted
}

Production


providers {
  auth      = "google"    # Google OAuth
  workspace = "google"    # Google Workspace
  search    = "algolia"   # Managed cloud
}

Or mix: Okta auth + Google workspace + Meilisearch

Common Questions

Q: Why multiple workspace providers?

A: Flexibility and risk mitigation. Organizations may have Google, Microsoft, or neither. Local workspace enables air-gapped deployments and cost-sensitive environments.

Q: Performance overhead from abstraction?

A: Minimal. Provider interfaces add <5ms overhead (interface method calls). Negligible compared to network latency (50-200ms for cloud APIs). The flexibility gains far outweigh the cost.

Q: Maturity of local workspace?

A: Production-ready. ADR-071 and ADR-074 document the design. E2E tests validate all workflows. Used daily in development (eating our own dog food).

Q: Can we migrate existing documents?

A: Yes. The migration pipeline preserves all metadata, versions, and relationships. Command: hermes migrate --source google --target local (or vice versa). Design documented in RFC-080.

Q: Total cost of ownership?

A: Lower for self-hosted. Meilisearch eliminates Algolia costs. Local workspace eliminates Google API call costs. Trade-off: self-hosting infrastructure (PostgreSQL, Meilisearch). But you gain full control and predictable costs.

Q: When can we deploy in production?

A: Today for local/development use cases. Google Workspace + Algolia is production-proven. Office365 provider arriving Q1 2025. All provider combinations are tested and documented.
